  <name>Task 1: Create base.html with sidebar and shared structure</name>
  <files>src/lattice/web/templates/base.html</files>
  <action>
Create `base.html` as a new Jinja2 base template. Extract the shared structure from the existing templates into this single source of truth.

**Structure:**
1. Full `<!DOCTYPE html>` through `</html>` skeleton
2. `<html lang="en" class="dark">` (theme default)
3. `<head>` containing:
   - Meta charset and viewport
   - `<title>LATTICE // {% block title %}{% endblock %}</title>`
   - Tailwind CDN script tag
   - Google Fonts preconnect + Orbitron/Space Mono link
   - `<link rel="stylesheet" href="/static/css/styles.css?v=10">` (bump from v=9)
   - `{% block head_extra %}{% endblock %}` for page-specific styles or scripts (like D3.js)
4. `<body class="font-mono {% block body_class %}{% endblock %}">` for page-specific body classes
5. **Sidebar navigation** (`<nav class="sidebar-rail">`):
   - Graph icon link: `<a href="/" class="sidebar-icon {% if current_page == 'graph' %}active{% endif %}" title="Graph">` with the lattice logo SVG (4 connected squares, simplified to 24x24 viewBox, 20x20 size)
   - Active Runs icon link: `<a href="/runs" class="sidebar-icon {% if current_page == 'runs' %}active{% endif %}" title="Active Runs">` with activity/pulse SVG polyline
   - Run History icon link: `<a href="/history" class="sidebar-icon {% if current_page == 'history' %}active{% endif %}" title="Run History">` with clock SVG (circle + polyline hands)
   - Use the exact SVG icons from the research doc (see 04-RESEARCH.md Example 2)
6. `<main class="main-content {% block main_class %}{% endblock %}">{% block content %}{% endblock %}</main>`
7. Corner decorations (4 divs: top-left, top-right, bottom-left, bottom-right with class "corner-decor")
8. Theme toggle script (consolidate from existing templates):
   - Read `localStorage.getItem('theme')`, apply 'light' class if saved
   - NO theme toggle button in base.html — that stays in per-page headers (some pages have it, asset_live has it in a different spot)
   - Just the initialization script: read localStorage, set html class, no button listener here
9. `{% block scripts %}{% endblock %}` for page-specific JS

**Important details:**
- The sidebar `<nav>` is OUTSIDE the `<main>` content area (sibling, not parent)
- The sidebar icons each need `position: relative` (set in CSS) for the CSS tooltip to position correctly
- Do NOT include any page header in base.html — each page defines its own header inside `{% block content %}` (the header varies too much between pages to share)
- The `/runs` link for Active Runs will 404 until Phase 5 creates the route — this is acceptable per research recommendation
