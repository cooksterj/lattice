**index.html (graph page):**

Rewrite index.html to extend base.html. This is the most layout-sensitive page because the D3 graph SVG fills the viewport.

Structure:
```
{% extends "base.html" %}
{% block title %}Asset Graph{% endblock %}
{% block head_extra %}
<script src="https://d3js.org/d3.v7.min.js"></script>
{% endblock %}
{% block content %}
  [entire content of the current <div class="flex h-screen relative z-10"> through </aside>]
  [tooltip div]
{% endblock %}
{% block scripts %}
<script src="/static/js/graph.js?v=16"></script>
{% endblock %}
```

Key changes:
- Remove the entire `<!DOCTYPE>` through `<body>` opening tags (provided by base.html)
- Remove the Tailwind CDN, Google Fonts, and styles.css links (provided by base.html)
- KEEP the D3.js script tag — move to `{% block head_extra %}`
- KEEP the full header (with logo, nav links, node count, relayout, theme toggle) inside `{% block content %}` — the header is specific to the graph page
- KEEP the right-side asset detail sidebar (`<aside id="sidebar">`) — this is the graph's detail panel, not the navigation sidebar
- KEEP the tooltip div
- Remove corner-decor divs (provided by base.html)
- Remove `</body></html>` tags (provided by base.html)
- The graph page's existing header has its own theme toggle button — KEEP it, the base.html only does theme initialization (reading localStorage), not the toggle button

**CRITICAL for graph layout:** The graph container is currently `<div class="flex h-screen relative z-10">`. With the sidebar's `margin-left: 52px` on `.main-content`, the graph area will be narrower. The `LatticeGraph` class uses `container.clientWidth` which will automatically account for this. But verify the header (which is `absolute top-0 left-0 right-0`) still positions correctly within the `.main-content` area. If the header uses `left-0 right-0`, it will fill the main-content width (which is correct since it's inside `<main class="main-content">`).
