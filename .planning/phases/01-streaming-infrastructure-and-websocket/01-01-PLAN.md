```python
self._asset_subscribers: dict[str, set[WebSocket]] = {}

```

Add constant for replay buffer size as class-level attribute:
```python
REPLAY_BUFFER_SIZE: int = 500
```

Add these methods to ExecutionManager:

1. `add_asset_subscriber(self, asset_key: str, ws: WebSocket) -> None`:
   - If asset_key not in `_asset_subscribers`, create empty set.
   - Add ws to the set.
   - Log at debug level: "Asset subscriber added for %s, total: %d"

2. `remove_asset_subscriber(self, asset_key: str, ws: WebSocket) -> None`:
   - Discard ws from `_asset_subscribers[asset_key]` if key exists.
   - If set is empty after removal, delete the key from the dict.
   - Log at debug level.

3. `async def broadcast_to_asset(self, asset_key: str, message: dict[str, Any]) -> None`:
   - Take a snapshot of subscribers: `subscribers = set(self._asset_subscribers.get(asset_key, ()))`.
   - Iterate and call `await ws.send_json(message)` in try/except.
   - Collect dead sockets and remove them from `_asset_subscribers[asset_key]` after iteration.
   - Follow the exact same pattern as the existing `broadcast()` method.

4. `def get_replay_buffer(self, asset_key: str) -> list[dict[str, Any]]`:
   - Return `list(self._replay_buffers.get(asset_key, deque()))`.

Add tests to a new `TestAssetSubscriberRegistry` class in test_web.py:

1. `test_add_asset_subscriber` -- Add a mock WS, verify it's tracked.

3. `test_remove_nonexistent_subscriber_no_error` -- Remove from empty key, no error.
4. `test_broadcast_to_asset_sends_to_subscribers` (async) -- Add 2 mock WSs for same key, broadcast, verify both received the message.
5. `test_broadcast_to_asset_only_targets_key` (async) -- Add WS to "asset_a" and "asset_b", broadcast to "asset_a", verify only asset_a's WS received.
6. `test_broadcast_to_asset_removes_dead_sockets` (async) -- Add 2 mock WSs, make one raise Exception on send_json, broadcast, verify dead one removed and live one still tracked.
7. `test_get_replay_buffer_empty` -- No buffer for key, returns empty list.

Use `unittest.mock.AsyncMock` for mock WebSockets. The mock WS needs `send_json` as an AsyncMock.
