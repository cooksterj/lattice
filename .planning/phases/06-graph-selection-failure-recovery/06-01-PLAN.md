<task type="auto">
  <name>Task 1: Implement graph selection and context-aware Execute button</name>
  <files>src/lattice/web/static/js/graph.js</files>
  <action>
Make the following changes to graph.js. The research doc (06-RESEARCH.md) has detailed code examples for each change.

**1. Replace node click handler (line ~526-530):**
Change the `.on('click')` handler in `setupEventListeners()` from calling `this.openAssetWindow(d.id)` to calling `this.handleNodeClick(d)`. Keep the `event.defaultPrevented` guard and `event.stopPropagation()`.

**2. Add three new methods after `selectNode()` (after ~line 792):**

`handleNodeClick(d)` -- Toggle selection: if clicking the already-selected node (`this.selectedNode && this.selectedNode.id === d.id`), call `this.deselectNode()`. Otherwise call `this.selectNodeForExecution(d)`.

`selectNodeForExecution(d)` -- Set `this.selectedNode = d`. Apply the `.selected` CSS class via `this.nodeElements.classed('selected', n => n.id === d.id)`. Call `this.updateExecuteButtonLabel()`.

`deselectNode()` -- Set `this.selectedNode = null`. Remove `.selected` class from all nodes via `this.nodeElements.classed('selected', false)`. Call `this.updateExecuteButtonLabel()`. Also close the right sidebar (`sidebar.classList.add('translate-x-full')`) and remove `sidebar-open` from execution controls -- replicate the existing deselection behavior from the SVG background click handler.

`updateExecuteButtonLabel()` -- Get the execute button and its span child. If `this.selectedNode` is set: get the node name via `this.selectedNode.name || this.selectedNode.id`, get status from `this.executionState.assetStatuses.get(this.selectedNode.id)`. If status is `'failed'`, set span text to `RE-EXECUTE FROM ${name.toUpperCase()}`. Otherwise set to `EXECUTE FROM ${name.toUpperCase()}`. If no selectedNode, set to `'EXECUTE'`. Guard against button not existing (early return if `!btn`).

**3. Modify SVG background click handler (line ~543-550):**
Replace the inline deselection logic with a call to `this.deselectNode()`. The existing handler already clears selectedNode and removes the selected class -- just delegate to the new method. Remove the direct sidebar/selectedNode/nodeElements manipulation since `deselectNode()` handles it.

**4. Modify closeSidebar click handler (line ~533-540):**
Similarly, replace the inline deselection logic with `this.deselectNode()`.

**5. Add Escape key listener in `setupEventListeners()`:**
Add `document.addEventListener('keydown', (event) => { if (event.key === 'Escape' && this.selectedNode) { this.deselectNode(); } });` at the end of setupEventListeners.

**6. Modify `startExecution()` (line ~1013-1028):**
After the date parameter construction and before the fetch call, add: if `this.selectedNode` is set, add `requestBody.target = this.selectedNode.id` and `requestBody.include_downstream = true` to the request body. This passes the targeted execution parameters to the existing backend API which already supports them.

**7. Modify `startExecution()` progress total (line ~999):**
When `this.selectedNode` is set, set the progress total text to `'...'` instead of `this.nodes.length`. The `showExecutionComplete()` method already corrects the total with the actual count from the server response. This avoids showing a misleading "0/15" when only 3 assets are being executed.

**8. Modify `stopExecution()` (line ~1206-1229):**
Replace the hardcoded `btn.querySelector('span').textContent = 'EXECUTE'` with a call to `this.deselectNode()`. This clears selection AND resets the button label (since `deselectNode` calls `updateExecuteButtonLabel` which sets label to 'EXECUTE' when no node selected). Do NOT call `deselectNode()` before clearing `isRunning` -- the order should be: set `isRunning = false`, close WebSocket, then `this.deselectNode()`, then handle button enabled/running states. Actually, be careful: `deselectNode()` sets selectedNode to null and updates button label. Then we still need `btn.disabled = false` and `btn.classList.remove('running')`. So call `this.deselectNode()` AFTER the btn state changes, or simply: set isRunning=false, close WS, set btn.disabled=false, remove 'running' class, call deselectNode() (which sets label to EXECUTE and clears selection).

**9. Modify `updateAssetStatus()` (line ~1145):**
After setting the status in the Map and updating the node class, check if the updated asset is the currently selected node: `if (this.selectedNode && this.selectedNode.id === assetId)`. If so, call `this.updateExecuteButtonLabel()`. This handles Pitfall 2 from the research -- the button label dynamically updates as execution status changes (e.g., from "RE-EXECUTE FROM X" to "EXECUTE FROM X" when X transitions from failed to running/completed).

**10. Modify `startExecution()` -- reset node visual states (line ~995):**
The line `this.nodeElements.attr('class', 'node')` resets ALL node classes including `.selected`. After this line, if `this.selectedNode` is set, re-apply the selected class: `if (this.selectedNode) { this.nodeElements.classed('selected', n => n.id === this.selectedNode.id); }`. This keeps the selected node highlighted during execution.

**What NOT to do:**
- Do NOT remove `openAssetWindow()`, `showPopupBlockedNotice()`, `assetWindows`, or `window.name` -- that cleanup belongs to Phase 7.
- Do NOT open the right-side detail sidebar on node click -- selection is purely visual (highlight + button label).
- Do NOT compute downstream blast radius client-side -- just highlight the single selected node.
- Do NOT prevent selection of non-failed nodes -- ALL nodes are selectable per success criteria.
