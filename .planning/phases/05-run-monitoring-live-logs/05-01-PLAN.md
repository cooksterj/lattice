**3. JavaScript implementation (inside `{% block scripts %}`):**

Follow the dual-mode pattern from research:

```javascript
const STATES = { LOADING: 'loading', LIVE: 'live', IDLE: 'idle' };
let currentMode = STATES.LOADING;
let ws = null;
let reconnectTimer = null;
let assetStatuses = {}; // key -> {status, started_at, duration_ms, error}
let totalAssets = 0;

async function initialize() {
    connectWebSocket();
    const status = await fetchJSON('/api/execution/status');
    if (status.is_running) {
        enterLiveMode(status);
    } else {
        enterIdleMode();
    }
}
```

Key JavaScript behaviors:

a) **connectWebSocket()**: Connect to `/ws/execution` (same endpoint the graph page uses). Handle messages:
   - `asset_start`: Add/update asset row to "running" status. If in idle mode, transition to live mode.
   - `asset_complete`: Update asset row to "completed" or "failed". Update progress counter.
   - `execution_complete`: Transition to idle mode. Fetch last run summary.
   - `memory_update`: Ignore (not needed on this page).
   - `partition_start`/`partition_complete`/`checks_complete`: Optionally show info, but not critical.

b) **enterLiveMode(status)**: Show live-panel, hide idle-panel. Populate initial asset statuses from `status.asset_statuses`. Set mode badge to "LIVE" with pulsing animation.

c) **enterIdleMode()**: Show idle-panel, hide live-panel. Fetch `GET /api/history/runs?limit=1` and render last run summary card. Set mode badge to "IDLE".

d) **Asset row rendering**: Each asset row is an `<a>` element linking to `/asset/{key}/live`. Use `textContent` for all user-provided strings (XSS safety). Status shown with color-coded badge (running=yellow with pulse, completed=cyan, failed=pink, pending=gray).

e) **Progress bar**: Shows `{completed + failed} / {total} ASSETS` and a visual bar fill.

f) **Last run summary card** (idle mode): Shows run_id, status (completed/failed), duration, asset counts, timestamp. If no runs exist, show "NO PREVIOUS RUNS" message.

g) **fetchJSON helper**: Simple async fetch wrapper returning parsed JSON.

h) **Theme toggle**: Same pattern as index.html and other pages.

i) **Reconnection**: On WebSocket close, reconnect after 2s. On reconnect, re-fetch REST status to re-sync.
